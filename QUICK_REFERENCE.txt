# FreelancerChatProtection Integration - Quick Reference

## What Was Integrated?

### ✅ Complete Integration Done

1. **OCR Service (FreelancerChatProtection)** - Standalone FastAPI service
   - Image text extraction via Tesseract
   - PDF text processing
   - Text file reading
   - Real-time text enhancement
   - Comprehensive error handling

2. **Backend Integration** - Django chat system
   - Message analysis with content detection
   - Image attachment OCR extraction
   - Restricted content flagging
   - Status tracking and error reporting

3. **Multi-Service Architecture**
   - Backend API: `http://localhost:8000`
   - Protection Service: `http://localhost:8001`
   - Frontend: `http://localhost:3000`
   - MongoDB Database: `mongodb://localhost:27017`

```bash
# Start services in background
```

## API Endpoints

### Chat Analysis
```
POST http://localhost:8000/api/chat/analyze/
```
**Body:**
```json
{
  "content": "Message text",
  "file": "optional_image_file"
}
```

### Get Conversation
```
GET http://localhost:8000/api/chat/:contract_id/
```

### Send Message
```
POST http://localhost:8000/api/chat/messages/:contract_id/
```

### Image OCR (Direct)
```
POST http://localhost:8001/analyze-image
Form Data:
  - file: image file
  - enable_enhancement: true/false
```

### API Documentation
```
Backend:   http://localhost:8000/api/docs
Protection: http://localhost:8001/docs
```

## Configuration

### Backend `.env`
```env
MONGO_HOST=mongodb://root:rootpassword@mongodb:27017/
CHAT_PROTECTION_ENABLED=true
CHAT_PROTECTION_SERVICE_URL=http://protection:8001
```

### Protection Service
Uses default Tesseract path: `/usr/bin/tesseract` (configurable via env)

### Frontend `.env.local`
```env
NEXT_PUBLIC_API_URL=http://localhost:8000
NEXT_PUBLIC_PROTECTION_URL=http://localhost:8001
```

## System Requirements

### Minimum
- Python 3.9+
- Node.js 18+
- Docker & Docker Compose
- 2GB RAM
- 5GB disk space

### Recommended
- Python 3.11+
- Node.js 20+
- 4GB RAM
- 10GB disk space

### System Packages
- **Tesseract OCR** (required)
  - Windows: https://github.com/UB-Mannheim/tesseract/wiki
  - macOS: `brew install tesseract`
  - Linux: `sudo apt-get install tesseract-ocr`

## Troubleshooting Quick Fixes

### "Tesseract not found"
```bash
# Check installation
tesseract --version

# Set path in environment
export TESSERACT_CMD=/usr/bin/tesseract  # or windows path
```

### "Protection service not responding"
```bash
# Check if service is running
curl http://localhost:8001/docs

# View protection logs
docker-compose logs protection
```

### "Database connection failed"
```bash
# Check MongoDB
mongosh mongodb://root:rootpassword@localhost:27017/

# Or via Docker
docker-compose logs mongodb
```

### "Port already in use"
```bash
# Change port in docker-compose.yml or:
# Kill process on port
lsof -ti:8000 | xargs kill -9  # macOS/Linux
netstat -ano | findstr :8000   # Windows
```
